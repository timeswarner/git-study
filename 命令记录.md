# 第4课 配置作者信息
```
git config --global user.name sk
git config --global user.email songkan@163.commit
```
* 注意事项
  1. 在**家目录**使用`code .gitconfig`查看全局配置文件
  2. 查看某项目的配置，先cd到那个包，再在包里面使用`code .git/config`

* 查看已有的配置信息，也可以具体查看某一个属性
```
git config --list
git config username
```

* 配置默认的文本编辑器
```
git config --global core.editor code
```
* 新建一个文件夹`mkdir`
* 显示目录列表`ls`
* 显示详细详细列表`ll`
* 看当前在哪个文件夹`pwd`
* 回到上级目录`cd ..`

# 第5课仓库新建和维护旧仓库
1. 初始化仓库
```
git init
```
2. 复制某项目的仓库，在git根目录使用这个命令，根目录没有`.git`文件夹（如果某个文件夹有`.git`文件，就是一个仓库）
```
git clone https://github.com/***/**.git
```
3. 跑路命令
```
rm -rf *
```

# 第8课 版本库的忽略文件配置
在初始化版本库以后可以使用，忽略文件配置
```

``` 
***在git里面如果一个文件夹是空的，就不会被追踪***

# 第9课 从版本库删除文件资源的技巧
* 把本地文件和版本库里面都删除
```
git rm a.txt   
```
* 只删除版本库文件，保留本地文件
```
git rm --cached a.txt 
```
# 第10课 版本库中修改资料的名称
"修改文件名称a.php到b.php"

```
git mv a.php b.php
git commit -m 
```

# 第11课 使用log日志查看历史操作行为
  1.显示日志 `git log`

  2.显示文件的变动信息 `git log -p` 

  3.显示最近的一次提交 `git log -p -1` 

  4.简短信息 `git log --oneline` 

  5.文件的发生变化 `git log --name-only` 

  6.文件的具体变化 `git log --name-status`

# 第12课 试用amend修改最近一次提交事件
```

```
* 在已经完成一轮提交以后，再add一个文件，还可以使用amend命令
